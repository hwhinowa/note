유저 인터페이스를 만들기 위한 js 라이브러리

Angular, View는 완전한 기능을 갖추고 있는 프레임워크 React native는 모바일 네이티브 앱을 React로 만들 수 있는 프레임워크

Angular와 React는 서로 협력 관계 Angular2 native에서 React native를 사용하기로 했다고 한다 둘 중 어느 것이 더 좋다고 하기 어렵고 프레임워크과 라이브러리는 조금 다른 개념이며, 필요에 따라 선택하려 사용하는 것을 권장한다

Virtual DOM

React의 핵심 개념 기존 javascript에서는 DOM에 직접 접근하여 제어(getElement~)했는데, 이런 방법은 비효율적이고 성능도 저하되며 관리하기가 힘들다 Virtual DOM은 이것을 추상화한 것으로 javascript 객체에 불과하다 기존방식인 이미 렌더링된 html를 실제 DOM api를 사용하여 처리하는 것보다 빠르다 기존 프레임워크는 코드를 여러가지로 나누어 작업을 했지만(controller, directive, etc) React는 Component만을 사용함

React의 장점

  • 뛰어난 Garbage Collection, 메모리 관리, 성능
  • 서버사이드 & 클라이언트사이드 렌더링 지원(여기서 서버사이드는 초기구동 딜레이 & *SEO[검색엔진최적화])

    단, 서버사이드 자원을 사용하는 것은 감안해야 한다방문자가 많은 페이지인데 서버 컴퓨터가 좋지 않고 통신 분산 처리가 잘 되지 않았다면 서버사이드 렌더링은 비추천(인스타그램의 경우는 서버사이드 렌더링을 사용하는데 UI에 영향 없음)

  • 매우 간편한 UI 수정 및 재사용 (UI 컴포넌트화)
  • 페북 개발자들이 만들었고 계속 업데이트 중 (페북에서 일부 사용, 인스타그램은 모든 곳에서 사용)
  • 다른 프레임워크나 라이브러리와 혼용 가능

React의 단점

  • View only (데이터모델링, 라우팅, Ajax 등의 기능이 없음)
  • IE8 이하 지원 불가

    IE8 이하를 지원하려면 React v14 이하를 사용하고, 호환 가능하게 해주는 polyfill 사용해야 한다
